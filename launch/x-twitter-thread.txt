TWEET 2 (the problem):
Every long agent session hits the same wall: the context window fills up.

The usual answers are bad:
- Truncation → agent forgets what it was doing
- LLM summarization → costs a model call, adds latency, busts your prompt cache

I built a third option.

-------------------------------------------------------------------

TWEET 3 (the architecture):
Context Warp Drive folds old turns into compact structural skeletons.

It conserves exact identifiers (UUIDs, paths, SHAs, ports) in a "Coordinate Closet."

It freezes the folded prefix → byte-identical → provider cache stays hot.

It pages content back in when the agent touches a path again.

Zero model calls. Zero latency.
